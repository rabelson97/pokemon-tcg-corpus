# English Price Provider Comparison

Historical evaluation note: this document records an April 2026 provider audit for improving English Pokemon price coverage. It is not the live runtime contract. Current runtime behavior remains defined by [scripts/build_prices_db.py](/Users/rabelson/Documents/GitHub/pokemon-tcg-corpus/scripts/build_prices_db.py), [README.md](/Users/rabelson/Documents/GitHub/pokemon-tcg-corpus/README.md), and the CardHawk consumer code.

Choose the best path to ensure English cards show either a real USD market price or `Pending`, never a substituted EUR Cardmarket price.

## Recommendation

### Immediate safe product behavior

Change English pricing to:

`PokemonTCG.io tcgplayer/USD -> Pending`

Do not publish `cardmarket/EUR` for English cards.

This is the lowest-risk fix because it removes false market substitution immediately without changing providers.

### Recommended provider path

Primary target:

- `TCGplayer official API`

Fallback order after access is obtained and mapped:

1. `TCGplayer official API`
2. `PokemonTCG.io tcgplayer/USD` only when the card mapping is exact and the row is fresh
3. `Pending`

Do not add these as automatic silent fallbacks in v1:

- `PriceCharting`
- `eBay sold/comps`

Reasons:

- `PriceCharting` is a useful valuation source, but it is not the same market signal as TCGplayer market price and has a tighter paid-access model.
- `eBay` is viable only if you are willing to own a full comp-normalization pipeline, which is significantly more engineering than the current builder.

## Why Collectr And HoloDex Don’t Change The Recommendation

Public materials do not clearly identify an official provider stack for either app, so they are not sufficient as an implementation dependency decision.

- `Collectr`: public materials suggest marketplace/sold-listing style features, but the exact backend price providers were not verified from official docs during this audit.
- `HoloDex`: public materials describe real-time price tracking and recent sales, but the exact provider mix was not publicly named in the sources reviewed.

These apps are useful as competitive references, not as authoritative integration guidance.

## Concrete Implementation Scope

### `github/pokemon-tcg-corpus`

1. Update [scripts/build_prices_db.py](/Users/rabelson/Documents/GitHub/pokemon-tcg-corpus/scripts/build_prices_db.py) so English cards never publish `cardmarket/EUR`.
2. Keep non-English `cardmarket` behavior unchanged.
3. Preserve row provenance in `prices.db`:
   - `source_name`
   - `market_code`
   - `currency_code`
   - `updated_at`
4. Expand `prices-build-summary.json` to emit:
   - English `eur_only` count
   - English `missing` count
   - by-set diagnostics for missing USD
   - a small sample of affected card IDs
5. Add a provider abstraction only when a second real USD source is ready to integrate.

### `github/CardHawk`

1. Keep the existing USD preference logic in [DatabaseStores.kt](/Users/rabelson/Documents/GitHub/CardHawk/shared/src/commonMain/kotlin/com/cardhawk/shared/data/database/DatabaseStores.kt).
2. Rely on the DB contract change so English cards without USD render as `Pending`.
3. Do not synthesize or convert EUR to USD in the app.

## Acceptance Criteria

- English cards with no valid USD source do not have a published price row.
- Recent Discoveries never show `EUR` for an English card id.
- Existing English cards with valid `tcgplayer/USD` remain unchanged.
- Non-English locales continue to use their locale-appropriate rows.
- Build artifacts expose enough diagnostics to explain future USD coverage gaps without manual SQLite inspection.
